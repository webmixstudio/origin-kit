# Security Policy

Thank you for helping keep **OriginKit** secure.

We take the security of OriginKit seriously and appreciate the efforts of security researchers and community members who responsibly disclose vulnerabilities.

## Supported Versions

OriginKit is currently in active development and has **not yet reached its first stable release**.

At this stage, security fixes are applied directly to the `main` branch. Once the project reaches **v1.0.0**, security updates will follow a version support policy.

| Version         |      Supported      |
| :-------------- | :-----------------: |
| `main`          |          ✅          |
| Stable releases | *Not available yet* |

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability, **please do not create a public GitHub issue**.

Instead, report it privately by emailing:

**[kuldeep@webmixstudio.com](mailto:kuldeep@webmixstudio.com)**

Please include, where possible:

* A clear description of the vulnerability
* Steps to reproduce the issue
* The potential security impact
* Proof of concept or reproduction details (if applicable)
* Any suggested remediation

## Response Process

We aim to:

* Acknowledge reports within **3 business days**
* Validate and assess reported vulnerabilities
* Keep reporters informed throughout the investigation
* Release a fix as soon as reasonably possible based on severity

After a fix has been released, the vulnerability may be publicly disclosed through the project's release notes or GitHub Security Advisories when appropriate.

## Responsible Disclosure

To help protect OriginKit users, we kindly request that you:

* Do not publicly disclose a vulnerability before a fix has been released.
* Avoid accessing, modifying, or deleting data that does not belong to you.
* Make a good-faith effort to avoid disrupting the availability, integrity, or confidentiality of the project or its users.

We appreciate your efforts in helping improve the security of OriginKit.
